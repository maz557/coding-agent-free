مساعد برمجي تفاعلي يعمل بالذكاء الاصطناعي يعمل في **الطرفية** أو **متصفح الويب** — مدعوم بواجهات برمجة تطبيقات سحابية **مجانية** (OpenRouter، Groq، Google، DeepSeek، Mistral) ونماذج **محلية** (Ollama، LM Studio، Llama.cpp). يقوم بقراءة الملفات وكتابتها والبحث فيها ونسخها ونقلها وحذفها، بالإضافة إلى تنفيذ أوامر الصدفة — كل ذلك عبر استدعاء الأدوات باللغة الطبيعية.

> 💡 **جاهز للعمل دون اتصال**: مع خادم محلي، يعمل الوكيل بالكامل دون اتصال بالإنترنت — لا حاجة للإنترنت، ولا تغادر بياناتك جهازك.

## 🧠 لماذا هذا الوكيل؟

| المشكلة | كيف يحلها هذا الوكيل |
|---------|--------------------------|
| تكلفة مساعدي البرمجة 20 دولارًا شهريًا (ChatGPT+, Claude Pro) | **مجاني 100%** — يستخدم الطبقة المجانية من OpenRouter وGroq وGoogle وDeepSeek وMistral + النماذج المحلية |
| توقف أحد المزودين / فرض حدود على المعدل | **8 مزودين** — تحويل تلقائي عند 429 + يدوي عبر `/model <n>` |
| لا يوجد اتصال بالإنترنت / منطقة مقيدة | **نماذج محلية** (Ollama، LM Studio، Llama.cpp) — بدون اتصال بالكامل |
| مخاوف الخصوصية مع واجهات السحابية | تشغيل **نماذج محلية فقط** — لا تغادر أي بيانات جهازك |
| الإعداد معقد جدًا | **`npm run setup`** — معالج تفاعلي، لا حاجة لتحرير `.env` يدويًا |
| الذكاء الاصطناعي ينفذ أوامر خطيرة | **الوضع الآمن** (`/safe`) — أوامر الصدفة المسموح بها فقط |
| الوكيل يعلق في حلقات تكرارية | **كشف ذكي** — يتوقف بعد 3× استدعاءات متطابقة للأدوات |
| المزود يفرض حدًا على المعدل | **تحويل تلقائي** — يبدل المزود تلقائيًا عند 429 |
| نتائج الأدوات الطويلة تهدر الرموز | **ضغط الرموز المميزة** — اقتطاع البداية والنهاية + إزالة التكرارات |
| تريد استخدام بيئة التطوير الخاصة بك مع نماذج مجانية | **واجهة برمجة متوافقة مع OpenAI** — `npm run setup-ide` يربط Cline وContinue.dev وCursor |

## بداية سريعة

```bash
git clone https://github.com/maz557/coding-agent-free.git
cd coding-agent-free
npm install
```

شغّل معالج الإعداد التفاعلي (موصى به):
```bash
npm run setup
```

أو أنشئ `.env` يدويًا (اختر مزودًا واحدًا على الأقل):
```bash
# OpenRouter (الأسهل — مفتاح واحد لأكثر من 18 نموذجًا مجانيًا لاستدعاء الأدوات)
echo "OPENROUTER_API_KEY=sk-or-v1-..." >> .env

# المزودون الاختياريون:
echo "GROQ_API_KEY=gsk_..." >> .env      # استدلال فائق السرعة
echo "GOOGLE_API_KEY=AIza..." >> .env     # نماذج Gemini
echo "DEEPSEEK_API_KEY=sk-..." >> .env    # DeepSeek
echo "MISTRAL_API_KEY=..." >> .env        # نماذج Mistral
```

**بدون اتصال بالإنترنت بالكامل (لا حاجة لمفاتيح API):**
```bash
# 1. استنساخ وتثبيت (يتطلب اتصالاً بالإنترنت مرة واحدة)
git clone https://github.com/maz557/coding-agent-free.git
cd coding-agent-free
npm install

# 2. شغّل خادم النموذج المحلي (اختر واحدًا):
#    ollama run qwen3:14b           # Ollama
#    # أو شغّل LM Studio / llama-server

# 3. شغل الوكيل:
npm start

# 4. داخل الوكيل، أضف نموذجك المحلي:
#    أنت: /add 6 ollama:auto
#    أنت: /model 6
```

## واجهة CLI

```bash
npm start
```

> في ويندوز، انقر نقرًا مزدوجًا على `run-cli.bat`.
> 📝 **اللغات من اليمين إلى اليسار (الفارسية، العربية، الأردية، العبرية، إلخ):** إذا كانت شاشة الطرفية تعرض النص من اليمين إلى اليسار بشكل غير صحيح، استخدم `run-cli-rtl.bat` بدلاً من ذلك — فهو يُطلق عبر [WezTerm](https://wezfurlong.org/wezterm/) مع دعم ثنائي الاتجاه المناسب.
> **دعم العربية (RTL):** توفر الواجهة دعمًا للغات التي تُكتب من اليمين إلى اليسار من خلال WezTerm؛ استخدم الملف `run-cli-rtl.bat` لعرض النص العربي بشكل صحيح في الطرفية.

## الواجهة الويب

```bash
npm run web
# افتح http://localhost:3000 في متصفحك
```

> في ويندوز، انقر نقرًا مزدوجًا على `run-web.bat`.

تدعم واجهة الويب نفس ميزات الطرفية — الردود المتدفقة، استدعاءات الأدوات، تبديل النموذج (جميع المزودين الثمانية + الإعدادات المسبقة للمستخدم)، تبديل الوضع الآمن، مسار السماح، وإعادة تعيين المحادثة. علامات تبويب متعددة في المتصفح مدعومة بجلسات مستقلة. تشترك واجهة CLI والويب في نفس تكوين النموذج (`src/config/models.ts`) ومحرك الأدوات (`fileManager.ts`).

يكشف خادم الويب أيضًا عن **واجهة برمجة تطبيقات متوافقة مع OpenAI** على `http://localhost:3000/v1/chat/completions`، بحيث يمكن لأي عميل متوافق مع OpenAI (Cline، Continue.dev، Cursor، إلخ) استخدام المزودين المهيئين لديك من خلال نقطة نهاية واحدة مع دعم التحويل التلقائي.

قم بتكوين بيئة التطوير الخاصة بك تلقائيًا:
```bash
npm run setup-ide
```

سيؤدي هذا إلى تكوين **Cline** و**Continue.dev** و**Cursor** لتوجيهها إلى وكيل API المحلي، باستخدام مفاتيح المزودين من ملف `.env` الخاص بك عبر نفس التوجيه.

**جلسة مثال:**

```
┌──────────────────────────────────────────────────────────┐
│  💬 Coding Agent                       [🔒 آمن] [🌐 P]  │
│  ──────────────────────────────────────────────────────── │
│                                                          │
│  أنت: create a hello.py that prints "Hello from Web UI"  │
│                                                          │
│  ⏳ تفكير... [النموذج: openrouter/free]                  │
│                                                          │
│  🛠  write_file(path="hello.py", content="...")          │
│  ✔  تمت كتابة الملف (25 بايت)                            │
│                                                          │
│  🛠  run_command(command="python hello.py")              │
│  ✔  Hello from Web UI                                    │
│                                                          │
│  تم! تم إنشاء hello.py والتحقق من طباعته:               │
│  Hello from Web UI                                       │
│  ──────────────────────────────────────────────────────── │
│ [إدخال الرسالة...                           ] [إرسال]   │
└──────────────────────────────────────────────────────────┘
```

> تتيح لك قائمة النماذج المنسدلة التبديل بين جميع المزودين الثمانية وإعداداتك المحفوظة أثناء الجلسة. تظهر الردود المتدفقة رمزًا تلو الآخر في الوقت الفعلي.
> انقر على زر **`?`** في الرأس لفتح نافذة تعليمات تحتوي على دليل الاستخدام وتعليمات تبديل النموذج وأوامر CLI المكافئة.

## أمثلة تفاعلية

**"أنشئ سكريبت بايثون يطبع أرقام فيبوناتشي"**

سينشئ الوكيل الملف ويكتب الكود ثم يشغله للتحقق:

```
أنت: write a fibonacci.py that prints first 20 numbers
⏳ تفكير...
  🔧 write_file({"path":"fibonacci.py","content":"..."})
  🔧 run_command({"command":"python fibonacci.py"})
الوكيل: تم! تم إنشاء fibonacci.py والتحقق من المخرجات: 0, 1, 1, 2, 3, 5...
```

**"ابحث عن جميع ملفات TypeScript التي تستدعي fetch() واستبدلها بـ axios"**

```
أنت: find all .ts files with fetch() calls and change them to axios
  🔧 search_content({"pattern":"fetch(","filePattern":"*.ts"})
  🔧 read_file({"path":"src/api.ts"})
  🔧 replace_in_file({"path":"src/api.ts","old_str":"fetch(","new_str":"axios."})
الوكيل: تم تحديث 3 ملفات (api.ts, users.ts, auth.ts).
```

**"صحّح هذا الخطأ: Cannot read property 'map' of undefined"**

يقرأ الوكيل الملف المعني، ويحلل الكود، ويقترح ويطبق الإصلاح.

## الميزات

- **8 مزودين** — OpenRouter، Groq، Google، DeepSeek، Mistral + Ollama، LM Studio، Llama.cpp
- **5 إعدادات مسبقة مدمجة** — ابدأ بـ `openrouter/free` (يكتشف تلقائيًا النماذج المجانية العاملة)
- **إعدادات مسبقة للمستخدم** — احفظ/أضف/أزل نماذجك الخاصة باستخدام `/save` و`/add` و`/remove`
- **سلسلة احتياطية** — تحويل تلقائي عبر المزودين عند حد المعدل (429)، بالإضافة إلى احتياطيات على مستوى النموذج
- **13 أداة** — قراءة، كتابة، سرد (مع التفاصيل)، إنشاء مجلد، حذف ملف، حذف مجلد (بشكل متكرر)، إلحاق ملف، نسخ ملف، نقل ملف، معلومات الملف، بحث في المحتوى، استبدال في الملف، وتنفيذ أمر
- **ضغط الرموز المميزة** — اقتطاع بداية ونهاية نتائج الأدوات الطويلة + إزالة التكرارات التلقائية
- **نافذة منزلقة للسياق** — يحتفظ بآخر 20 تبادلاً افتراضيًا، ويقص تلقائيًا لتجنب أخطاء حد الرموز المميزة (قابل للتكوين عبر `MAX_EXCHANGES` / `MAX_TOOL_RESULT_LENGTH`)
- **كشف الحلقات الذكي** — يتوقف إذا تم استدعاء أداة 3+ مرات بشكل متطابق أو 5+ مرات متتالية
- **الوضع الآمن** (`--safe` / `/safe`) — أوامر الصدفة المسموح بها فقط
- **معالج الإعداد** — `npm run setup` يهيئ `.env` تفاعليًا
- **إعادة المحاولة التلقائية** — تراجع أسي + مهلة 120 ثانية (300 ثانية للنماذج المحلية)
- **التحقق من الصحة بـ Zod** — فحص أنواع وقت التشغيل لكل مدخلات ومخرجات الأداة
- **توحيد CLI والويب** — تكوين نموذج مشترك، ومطالبة النظام، ومحرك الأدوات، وتعريفات المزودين، والإعدادات المسبقة للمستخدم عبر كلا الواجهتين
- **opencode.json** — قواعد أذونات شاملة (أكثر من 100 نمط أمر آمن مسموح به تلقائيًا) ومخرجات أدوات مقتطعة لمطالبات أنظف
- **اقتطاع مخرجات الأدوات** — جميع نتائج الأدوات محددة بـ 5000 حرف (`MAX_TOOL_RESULT_LENGTH`) للحفاظ على سياق نظيف
- **استمرارية المحادثة** — حفظ/استعادة تلقائي للجلسات عبر عمليات إعادة التشغيل
- **تسجيل منظم** — عبر `pino` (stderr، لا يتداخل مع واجهة المستخدم)

## الأدوات المتاحة

| الأداة | الوصف |
|------|-------------|
| `read_file` | قراءة محتويات ملف |
| `write_file` | كتابة محتوى إلى ملف (إنشاء/استبدال) |
| `list_files` | سرد محتويات الدليل. استخدم `details:true` للحجم والطوابع الزمنية |
| `create_folder` | إنشاء مجلد جديد |
| `delete_file` | حذف ملف واحد |
| `delete_folder` | حذف مجلد. اضبط `recursive:true` للمجلدات غير الفارغة |
| `append_file` | إلحاق محتوى بملف موجود |
| `copy_file` | نسخ ملف من مصدر إلى وجهة |
| `move_file` | نقل أو إعادة تسمية ملف |
| `file_info` | الحصول على بيانات وصفية مفصلة (الحجم، الأذونات، الطوابع الزمنية) |
| `search_content` | البحث عن نص مطابق في الملفات. يدعم `filePattern` (مثل `*.ts`) و`maxResults` (الافتراضي 50). يتجاوز الملفات الأكبر من 1 ميغابايت |
| `replace_in_file` | استبدال أول ظهور لنص مطابق (حساس لحالة الأحرف) |
| `run_command` | تشغيل أمر صدفة في مساحة العمل |

## الأوامر

| الأمر | الوصف |
|---------|-------------|

| `/active` | عرض النموذج النشط الحالي |

## استخدام مزودين متعددين

كل إعداد مسبق مرتبط بمزود. التبديل بين الإعدادات المسبقة باستخدام `/model <n>` يعيد إنشاء عميل API تلقائيًا:

```
أنت: /add 6 groq:openai/gpt-oss-120b
✅ تمت إضافة الإعداد المسبق 6: [Groq] openai/gpt-oss-120b

أنت: /model 6
✅ تم التبديل إلى الإعداد المسبق 6: [Groq] openai/gpt-oss-120b
   (الآن يستخدم API Groq مع gpt-oss-120b)

أنت: /model 1
✅ تم التبديل إلى الإعداد المسبق 1: [OpenRouter] openrouter/free
   (العودة إلى OpenRouter)
```

### إضافة نماذج من مزودين آخرين

```
/add <n> <provider>:<model-id>
```

أمثلة:
```
/add 10 groq:llama-3.3-70b-versatile
/add 11 google:gemini-2.0-flash-exp
/add 12 deepseek:deepseek-chat
/add 13 mistral:codestral-latest
```

إذا حذفت المزود (مثل `/add 10 llama-3.3-70b-versatile`)، فسيتم تعيينه افتراضيًا إلى مزود الإعداد المسبق الحالي.

## النماذج المحلية (Ollama، LM Studio، Llama.cpp)

يدعم الوكيل أي خادم محلي متوافق مع OpenAI بدون أي تهيئة:

### بداية سريعة

تأكد من أن خادمك المحلي يعمل، ثم:

```
أنت: /add 6 ollama:auto
✅ تم اكتشاف النموذج تلقائيًا: llama3.2:latest
✅ تمت إضافة الإعداد المسبق 6: [Ollama] llama3.2:latest

أنت: /model 6
✅ تم التبديل إلى الإعداد المسبق 6: [Ollama] llama3.2:latest
```

أو لـ LM Studio:

```
أنت: /add 7 lmstudio:auto
✅ تمت إضافة الإعداد المسبق 7: [LM Studio] qwen2.5-coder-7b-instruct
```

تخبر الكلمة المفتاحية `:auto` الوكيل بالاتصال بالخادم المحلي واكتشاف النموذج المحمّل تلقائيًا.

### بداية سريعة — نموذج محلي محدد

```bash
# Ollama — اسحب واخدم نموذجًا لاستدعاء الأدوات
ollama pull llama3.2
ollama serve                  # يبدأ على المنفذ 11434

# Llama.cpp — اخدم نموذج GGUF مباشرة
llama-server -m qwen2.5-coder-1.5b-instruct-q4_k_m.gguf --port 8080

# LM Studio — يستخدم واجهة lms المدمجة
lms get llama-3.2-3b-instruct   # تحميل نموذج
lms load llama-3.2-3b-instruct  # تحميل في الذاكرة
lms server start --port 1234    # بدء خادم API
```

ثم أضف النموذج المحلي إلى الوكيل:
```
/add 6 ollama:auto
/add 7 lmstudio:auto
/add 8 llamacpp:auto
```

### المنافذ المخصصة

ضع في `.env`:
```bash
OLLAMA_HOST=http://localhost:11434/v1
LMSTUDIO_HOST=http://localhost:1234/v1
LLAMACPP_HOST=http://localhost:8080/v1
```

### المتطلبات

- **Ollama**: [تحميل](https://ollama.ai) → `ollama pull llama3.2` → `ollama serve`
- **LM Studio**: [تحميل](https://lmstudio.ai) → `lms get llama-3.2-3b-instruct` → `lms server start --port 1234`
- **Llama.cpp**: [تحميل](https://github.com/ggerganov/llama.cpp) → بناء أو الحصول على ملف ثنائي → `llama-server -m model.gguf --port 8080`
- يجب أن يدعم النموذج **استدعاء الأدوات** (استدعاء الدوال) لوظائف الوكيل الكاملة.
- لا حاجة لمفتاح API — يتم تخطي المزودين المحليين أثناء التحقق من صحة المفاتيح عند بدء التشغيل.
- جميع المزودين المحليين يستخدمون API المتوافق مع OpenAI، لذلك لا حاجة لحزم إضافية.

## مساحة العمل والأذونات

افتراضيًا، يمكن للوكيل الوصول فقط إلى الملفات داخل `./workspace`. للوصول إلى مسارات أخرى:

### الخيار 1: تغيير مساحة العمل الافتراضية (دائم)

عيّن `ALLOWED_DIR` في `.env`:
```
ALLOWED_DIR=.          # جذر المشروع — الوصول إلى كل شيء
ALLOWED_DIR=C:\path    # أي مسار مطلق
```

## استكشاف الأخطاء وإصلاحها

| الخطأ | السبب المحتمل | الإصلاح |
|-------|-------------|------|
| `403 Forbidden` | مفتاح API مفقود أو غير صالح | تحقق من أن `.env` يحتوي على المفتاح الصحيح لذلك المزود |
| `403 Forbidden` | قيود الإنترنت تحجب مضيف API | فعّل VPN/وكيل، عيّن `HTTPS_PROXY`، أو استخدم النماذج المحلية: `/add 6 ollama:auto` |
| `429 Rate limit exceeded` | الوصول للحد اليومي للطبقة المجانية | انتظر أو دع **التحويل التلقائي** يبدل المزود تلقائيًا. يدويًا: `/model <n>` |
| `Agent stopped: stuck detected` | نفس الأداة استُدعيت 3×+ على التوالي | يتم حقن رسالة استرداد تلقائيًا — أعد صياغة طلبك |
| `All 3 attempts failed` | النموذج غير متاح أو بطيء جدًا | جرب نموذجًا أصغر، استخدم النماذج المحلية، أو بدّل المزود بـ `/model <n>` |
| `tool_calls` مع وسائط فارغة | النموذج لا يدعم استدعاء الأدوات | استخدم نموذجًا مختلفًا |
| `ENOTFOUND` / `ECONNREFUSED` | قيود الإنترنت أو الحاجة لوكيل | فعّل VPN/وكيل، عيّن `HTTPS_PROXY`، أو استخدم النماذج المحلية |

## المشاريع الكبيرة

للمشاريع متوسطة إلى كبيرة الحجم، قد تؤدي النافذة المنزلقة الافتراضية (20 تبادلاً) إلى فقدان السياق الأقدم. زد هذه القيم في `.env`:

```env
# احتفظ بما يصل إلى 50 تبادلاً بين المستخدم والمساعد
MAX_EXCHANGES=50

# اسمح بنتائج أدوات تصل إلى 20,000 حرف
MAX_TOOL_RESULT_LENGTH=20000
```

يمكنك أيضًا إعادة تعيين المحادثة أثناء الجلسة باستخدام `/reset` إذا أصبح النموذج مشوشًا بسبب سياق قديم.

## القيود

- **الالتزام بمطالبة النظام**: قد تتجاهل بعض النماذج المجانية (مثل Nvidia Nemotron 550B) تعليمات النظام أو تتبعها جزئيًا. بدّل إلى نموذج آخر إذا لاحظت ذلك.
- **حدود المعدل**: مفاتيح API للطبقة المجانية لها حدود يومية (HTTP 429). يعيد الوكيل المحاولة مع تراجع أسي (بحد أقصى 3 محاولات)، لكن الحدود المستمرة تتطلب تبديل المزودين أو الانتظار.
- **نافذة الرموز المميزة**: مع نموذج سياق 128 ألف ونافذة منزلقة من 20 تبادلاً، قد تصل قواعد الأكواد الكبيرة إلى حدود السياق. زد `MAX_EXCHANGES` و`MAX_TOOL_RESULT_LENGTH` في `.env` للمشاريع الأكبر.
- **كشف التعليق**: يتوقف الوكيل بعد 3× استدعاءات متطابقة للأدوات أو 5× متتالية لنفس الاسم، ويحقن رسالة نظام استرداد، ويزيل آخر نتائج الأدوات. ما عليك سوى إعادة صياغة طلبك للمتابعة.
- **صدفة ويندوز**: قد تؤدي عوامل تشغيل خط أنابيب PowerShell (`|`، `&&`) إلى ظهور مطالبات أذونات مطولة تحت قواعد opencode.json الصارمة. الأوامر البسيطة تعمل دون مطالبات.
- **المسارات النسبية مقابل المطلقة**: تتعامل النماذج مع المسارات بشكل غير متسق — بعضها يستخدم مسارات نسبية والبعض الآخر مطلقة. يقوم الوكيل بتطبيع المسارات ضمن `ALLOWED_DIR`.

## الأمان

- جميع عمليات الملفات مقيدة بـ `ALLOWED_DIR` — `sanitizePath` يمنع هجمات اجتياز المسار
- يتم تنفيذ أوامر الصدفة داخل دليل مساحة العمل
- مفاتيح API مخزنة في `.env` (مدرج في `.gitignore`، لا يتم الالتزام بها أبدًا)
- الوضع الآمن (`/safe`) يقيد الأوامر بقائمة بيضاء
- الأوامر الخطيرة للصدفة محظورة بقائمة سوداء (rm -rf، dd، mkfs، wget، إلخ.)
- استخدم سكريبتات `local/` للنسخ الاحتياطي والاستعادة
